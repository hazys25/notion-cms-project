# 📋 제품 요구사항 정의서 (PRD)

> **프로젝트명:** 관심주식 뉴스 모음
> **문서 버전:** v1.0
> **최종 수정일:** 2026-06-14
> **문서 목적:** Notion 을 CMS 로 활용한 "관심주식 뉴스 모음" 웹 서비스의 제품 요구사항을 정의한다.

> ℹ️ **참고:** 현재 저장소는 Next.js 15 + Supabase 기반 스타터킷 상태이며, Notion 연동 코드(`@notionhq/client`)는 아직 포함되어 있지 않다.
> 본 문서가 정의하는 기능은 **신규 구현 대상**이다. 환경 변수명·라우트 경로 등 일부 세부값은 **제안(예시)** 이며 구현 단계에서 확정한다.

---

## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 프로젝트명 | 관심주식 뉴스 모음 |
| 한 줄 소개 | 관심 종목 관련 기사를 일자별로 모아 보여주는 뉴스 큐레이션 블로그 |
| 핵심 목적 | **전일 장마감 후 ~ 당일 장시작 전** 사이 관련 기사를 수집하여, 일자별 게시글로 작성·발행 |
| 작성 주기 | 매 거래일 1회 (장 시작 전 발행을 목표) |

### 1.1 CMS 로 Notion 을 선택한 이유

- 별도의 관리자(어드민) 화면을 직접 만들지 않고도, **Notion 에서 글을 작성하면 자동으로 블로그에 반영**된다.
- 비개발자도 익숙한 Notion 편집 환경에서 기사 정리·본문 작성을 할 수 있다.
- 데이터베이스(속성/태그/상태)를 통해 카테고리·태그·발행 상태를 구조적으로 관리할 수 있다.

> 💡 **비유:** Notion 을 "원고를 쓰는 작업실", 웹사이트를 "완성된 원고가 진열되는 서점"으로 보면 된다.
> 작업실(Notion)에서 글을 `발행됨` 상태로 바꾸면, 서점(웹) 진열대에 자동으로 책이 올라간다.

## 2. 목표 및 비목표 (Goals / Non-Goals)

### 2.1 목표 (Goals) — 이번 MVP 에서 달성한다

- ✅ Notion 데이터베이스를 단일 원본(Single Source of Truth)으로 사용해 글을 표시한다.
- ✅ 방문자가 글 목록 → 상세로 자연스럽게 탐색할 수 있다.
- ✅ 카테고리/검색으로 원하는 글을 빠르게 찾을 수 있다.
- ✅ 모바일·데스크톱 모두에서 읽기 편한 반응형 화면을 제공한다.

### 2.2 비목표 (Non-Goals) — 이번 MVP 범위에서 제외한다

- ❌ 기사 **자동 수집·크롤링** 자동화 (수집은 운영자가 수동으로 진행 후 Notion 에 정리)
- ❌ 사용자 로그인/회원/댓글 등 커뮤니티 기능
- ❌ 실시간 시세·차트 등 금융 데이터 연동
- ❌ 다국어(i18n) 지원

## 5. Notion 데이터베이스 구조

게시글 한 건은 Notion 데이터베이스의 한 페이지(row)에 대응한다.

| 속성명 (Property) | 타입 (Type) | 설명 |
|-------------------|-------------|------|
| `Title` | `title` | 게시글 제목 |
| `Category` | `select` | 카테고리 (단일 선택) |
| `Tags` | `multi_select` | 태그 (반도체, AI, 로봇 등 복수 선택) |
| `Published` | `date` | 기사 작성일 (목록 정렬·일자 표기 기준) |
| `Status` | `select` | 상태 — `초안` / `발행됨` |
| `Content` | `page content` | 본문 (Notion 페이지 본문 블록) |

> ⚠️ **발행 규칙:** 웹사이트에는 `Status = 발행됨` 인 글만 노출한다. `초안` 상태의 글은 목록·상세 어디에도 표시하지 않는다.

## 6. 화면 구성 (정보 구조)

| 화면 | 경로(제안) | 설명 |
|------|------------|------|
| 홈 | `/` | 발행된 글을 `Published` 기준 최신순으로 보여주는 목록 |
| 글 상세 | `/posts/[id]` | 개별 글의 본문(Notion content)을 렌더링 |
| 카테고리 | `/category/[category]` | 특정 카테고리에 속한 글 목록 |

```text
홈 (/)
 ├─ 글 카드 목록 (제목 · 카테고리 · 태그 · 작성일)
 │    └─ 카드 클릭 → 글 상세 (/posts/[id])
 ├─ 카테고리 필터 → 카테고리 목록 (/category/[category])
 └─ 검색창 → 키워드로 글 필터링
```

> 📌 위 라우트 경로(`/posts/[id]`, `/category/[category]`)는 **제안값**이며 구현 단계에서 확정한다.

## 7. MVP 범위

이번 MVP 에서 반드시 포함하는 항목:

- ✅ **Notion API 연동** — 데이터베이스에서 발행 글 조회 및 본문 렌더링
- ✅ **글 목록 및 상세 페이지** — 홈 목록 + 개별 글 상세
- ✅ **기본 스타일링** — Tailwind CSS + shadcn/ui 기반 읽기 좋은 레이아웃
- ✅ **반응형 디자인** — 모바일/데스크톱 대응

---

## 8. 구현 단계 (로드맵)

진행 상태는 ✅(완료) / 🚧(진행 중) / ⏳(예정) 로 표시한다.

| 단계 | 작업 | 상태 |
|------|------|------|
| 1 | **Notion API 패키지 설치 및 환경 설정** — `@notionhq/client` 설치, 클라이언트 초기화 | ⏳ |
| 2 | **Notion 데이터베이스 생성 및 API 키 설정** — DB 속성 구성, 통합(Integration) 키 발급·연결 | ⏳ |
| 3 | **글 목록 페이지 구현** — 발행 글 조회 → 최신순 카드 목록 | ⏳ |
| 4 | **글 상세 페이지 구현** — 페이지 본문(블록) 렌더링 | ⏳ |
| 5 | **스타일링 및 최적화** — 반응형/디자인 다듬기, 성능·캐싱 최적화 | ⏳ |

## 9. 환경 변수 (제안)

`.env.local` 에 다음 값을 설정한다. (실제 변수명은 구현 단계에서 확정)

| 변수 | 용도 |
|------|------|
| `NOTION_API_KEY` | Notion 통합(Integration) 시크릿 키 |
| `NOTION_DATABASE_ID` | 게시글이 저장된 Notion 데이터베이스 ID |

```bash
# .env.local 예시
NOTION_API_KEY=secret_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
NOTION_DATABASE_ID=xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

> 🔐 API 키는 서버 전용 환경 변수로 다루며, 클라이언트 번들에 노출되지 않도록 서버 측(서버 컴포넌트/라우트 핸들러)에서만 사용한다.

## 10. 향후 확장 (Out of MVP)

MVP 이후 단계에서 검토할 항목:

- ⏳ **기사 자동 수집/스케줄링** — 장마감·장시작 시점에 맞춘 자동 기사 수집 파이프라인
- ⏳ **발행 상태 Webhook 연동** — Notion 발행 시 즉시 사이트 갱신(ISR 재검증 트리거)
- ⏳ **태그 클라우드 / 인기 태그** — `Tags` 기반 탐색 강화
- ⏳ **RSS / 뉴스레터** — 일자별 게시글 구독 채널 제공
- ⏳ **OG 이미지·SEO 메타** 자동 생성으로 공유 최적화
